# Progress

Codeness 0.1.0 is implemented as a native SwiftUI/AppKit macOS application. Each exact selected workspace folder is represented by an ordinary `NSWindow`, with Codeness-owned Open Recent, duplicate focusing, and cross-launch restoration. Repository URLs are never attached to `NSDocument`, so AppKit has no document save path that can replace a repository; File > Save writes only Application Support metadata. A window owns one current activity and one implementer/reviewer session pair. Its initial main area configures a multiline Goal and independent Implement, Review, and Fix prompts copied from editable application defaults. Start Over archives the old activity under Application Support, discards its session IDs, and returns the same window to editable configuration with its previous Goal and prompts prefilled while preserving per-window model settings. The complete Goal is injected into every phase as a delimited reference block. Application preferences provide four model/reasoning defaults that are copied only into brand-new repository windows; every window then persists and edits its own independent selections. The Codex executable is an app-wide setting because all windows share one App Server.

The workflow now runs strict Implement, Review, and review-only Fix work units. Retry attempts remain inside the work unit whose phase they retry, and the next Implement starts only after Fix completes. A completed implementation still receives Review and Fix before the current activity completes. The relay provides conservative handoffs, phase classification, and concrete sidebar-safe labels. Its structured-output schema permits only dispositions valid for the already-known phase, repeats that validation locally, and retries malformed results automatically before requesting supervision. The detail view is reasoning-first and independently filters Reasoning, Actions, and Diagnostics; real-run protocol noise informed the recommended defaults, and legacy flat transcripts receive the same action filtering. A completed pass moves its exact final answer into an independently scrollable lower pane identifying it as the handoff source. Live selection advances only when the current run is selected and its transcript remains at the bottom. The shared Codex App Server explicitly inherits Codeness's environment and enables the user's shell profile for agent tool commands.

Interruption and recovery are durable: a run must be saved before Codex can start, a failed initial-activity save leaves a durable retry checkpoint, an accepted turn remains supervised even if its next checkpoint write fails, and completed Codex output must be saved before relay I/O can begin. Relay requests run outside the shared App Server event consumer; interrupted passes and pending handoffs resume without replaying completed repository edits; append-only transcripts are merged without replacing newer metadata with a stale log; and relay classifications that cannot advance expose retry/edit/override controls. Failed settings saves roll back in-memory changes, and failed loads never write placeholder data over unreadable history. Terminal notifications and App Server request-resolution events remain authoritative while windows close or requests auto-resolve, and concurrent requests are queued rather than replacing one another. App Server stdout is consumed by one ordered reader and drained through EOF before process exit is published. Approval sheets preserve the server's ordered `availableDecisions`, including structured execution-policy and network-policy responses.

Repository windows have a foreground-only graceful pause lifecycle. Closing or quitting steers active turns toward a coherent checkpoint, exposes an eager interrupt action, waits for authoritative terminal events, and saves an explicit recover/route/perform resume checkpoint before allowing the frontend to disappear. Reopening is passive until Resume reconnects the original Codex threads. All Codeness-owned state lives in Application Support, including the open-window registry, recents, full window/sidebar geometry, selected row, independent per-run transcript anchors, follow state, and workflow controls.
